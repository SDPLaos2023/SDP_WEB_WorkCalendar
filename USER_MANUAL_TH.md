# คู่มือการใช้งานระบบ Work Calendar (ฉบับภาษาไทย)

> **อัปเดตล่าสุด:** มีนาคม 2026

ระบบ **Work Calendar** เป็นเครื่องมือสำหรับวางแผน ติดตาม และประเมินผลการทำงานรายปีขององค์กร รองรับ 3 ภาษา (**ລາວ / ไทย / English**) สามารถเปลี่ยนภาษาได้ตลอดเวลาผ่านเมนูโปรไฟล์

---

## 1. คำศัพท์ที่ควรรู้ (Terminology)

| คำศัพท์ | ความหมาย |
|---|---|
| **Work Plan (แผนงาน)** | ชุดภารกิจรายปีของแต่ละแผนก สร้างโดยผู้จัดการ |
| **PROJECT (งานโครงการ)** | งานทำครั้งเดียว มีวันเริ่ม-สิ้นสุด วัดผลด้วย % ความสำเร็จ (0–100%) |
| **ROUTINE (งานประจำ)** | งานทำซ้ำรายวัน/สัปดาห์/เดือน วัดผลด้วย Compliance |
| **Compliance (การปฏิบัติตามแผน)** | % การส่งรายงานเทียบกับจำนวนรอบที่กำหนด |
| **Task Actual (บันทึกผลงาน)** | ข้อมูลที่เจ้าหน้าที่ส่งเข้าระบบในแต่ละรอบ |
| **Supervisor** | หัวหน้างานที่ถูก Manager มอบหมายให้ดูแลแผน |
| **Officer** | ผู้ปฏิบัติงาน — รับมอบงานจาก Supervisor |

---

## 2. การเข้าสู่ระบบ (Login)

1. เปิดเบราว์เซอร์และไปที่ URL ของระบบ
2. ใส่ **ชื่อผู้ใช้ (Username)** และ **รหัสผ่าน (Password)** ที่ Admin สร้างให้
3. กด **เข้าสู่ระบบ**

> **เปลี่ยนภาษา:** คลิกที่รูปโปรไฟล์มุมล่างซ้าย → เลือก **ภาษา** → เลือก ລາວ / ไทย / English

---

## 3. โครงสร้างเมนูหลัก

| เมนู | ใครใช้ได้ | ทำอะไร |
|---|---|---|
| 📊 **หน้าหลัก** | ทุก Role | ภาพรวมสถิติและกราฟความคืบหน้า |
| 📅 **แผนงาน** | ทุก Role | จัดการแผนงานและงานในแผน |
| 📁 **งานของฉัน** | ทุก Role | ดูและอัปเดตงานที่ได้รับมอบหมาย |
| 📈 **รายงาน** | ทุก Role | ดูและ Export รายงานหลากรูปแบบ |
| 👥 **ผู้ใช้ระบบ** | Admin / Manager | จัดการบัญชีและสิทธิ์ผู้ใช้ |
| 🏢 **แผนก** | Admin / Super Admin | จัดการโครงสร้างแผนก |
| 🌐 **บริษัท** | Super Admin | จัดการข้อมูลบริษัทในระบบ |

---

## 4. ขั้นตอนการใช้งาน (ตามลำดับ Role)

### ขั้นตอนที่ 1 — Super Admin / Admin: ตั้งค่าโครงสร้างองค์กร

> ⚠️ ต้องทำก่อน ถ้าไม่มีข้อมูลนี้ผู้ใช้คนอื่นจะไม่สามารถทำงานในระบบได้

1. ไปที่เมนู **บริษัท** (Super Admin เท่านั้น) → กด **สร้างใหม่** → กรอกชื่อบริษัทและรหัส
2. ไปที่เมนู **แผนก** → กด **สร้างใหม่** → เลือกบริษัทและกรอกชื่อแผนก
3. ไปที่เมนู **ผู้ใช้ระบบ** → กด **สร้างใหม่** → สร้างบัญชีผู้ใช้ และกำหนด Role ให้ถูกต้อง:

| Role | หน้าที่ |
|---|---|
| `ADMIN_COMPANY` | ดูแลภาพรวมของทั้งบริษัท |
| `MANAGER` | สร้างและจัดการแผนงานของแผนกตนเอง |
| `SUPERVISOR` | สร้างงานและมอบหมายงานให้ Officer |
| `OFFICER` | รับมอบงาน และส่งรายงานความคืบหน้า |

---

### ขั้นตอนที่ 2 — Manager: สร้างแผนงานประจำปี

1. ไปที่เมนู **แผนงาน**
2. กดปุ่ม **สร้างแผนใหม่** (ถ้าไม่มีแผนใดเลย ระบบจะแสดงข้อความแนะนำพร้อมปุ่มลัด)
3. กรอกข้อมูล: **ชื่อแผน, แผนก, ปี**
4. สถานะเริ่มต้นคือ **ฉบับร่าง (DRAFT)**

> 💡 แผนที่อยู่ในสถานะ DRAFT ยังไม่สามารถส่งงานได้

---

### ขั้นตอนที่ 3 — Manager / Supervisor: สร้างงานในแผน

1. คลิกเปิดแผนงานที่ต้องการ
2. กดปุ่ม **เพิ่มงาน**
   - เลือก **งานโครงการ (PROJECT)** → กำหนดวันเริ่ม-สิ้นสุด
   - เลือก **งานประจำ (ROUTINE)** → กำหนดความถี่ (รายวัน/สัปดาห์/เดือน) และช่วงเวลา
3. เลือก **ผู้รับผิดชอบ (Officer)** ที่จะมอบงานให้
4. กำหนด **ระดับความสำคัญ (Priority)**: LOW / MEDIUM / HIGH / CRITICAL

> 💡 ถ้ายังไม่มีงานในแผน ระบบจะแสดงข้อความแนะนำให้เพิ่มงานพร้อมปุ่มลัด

---

### ขั้นตอนที่ 4 — Manager: อนุมัติแผนงาน

1. เปิดแผนงาน → กดปุ่ม **อนุมัติและเริ่มใช้งาน**
2. สถานะจะเปลี่ยนเป็น **กำลังดำเนินการ (ACTIVE)**
3. เจ้าหน้าที่จะเห็นงานที่ถูกมอบหมายและสามารถส่งรายงานได้ทันที

> ⚠️ **สำคัญ:** เมื่อเปลี่ยนสถานะแล้วไม่สามารถย้อนกลับได้ เช่น `ACTIVE → CLOSED` ไม่สามารถกลับเป็น `ACTIVE` ได้

---

### ขั้นตอนที่ 5 — Officer: ติดตามและส่งรายงานงาน

1. ไปที่เมนู **งานของฉัน** → เลือก Tab **งานประจำ** หรือ **งานโครงการ**
2. ถ้ายังไม่มีงาน → ระบบจะแจ้ง _"ยังไม่ได้รับมอบหมายงานจาก Supervisor"_
3. คลิกที่การ์ดงาน → กดปุ่ม **รายงานผล** หรือ **อัปเดตความคืบหน้า**
4. กรอกข้อมูล:
   - **วันที่ปฏิบัติงาน (Actual Date)**
   - **% ความสำเร็จ** (สำหรับงานโครงการ)
   - **สถานะ:** สำเร็จ / สำเร็จบางส่วน / ไม่ได้ปฏิบัติ
   - **หมายเหตุ** (ถ้ามี)
5. กด **บันทึก**

> 💡 งานประจำ (Routine) ต้องส่งรายงานทุกรอบตามที่กำหนด มิฉะนั้นจะถูกนับเป็น **ขาด (Missed)**

---

## 5. การดูแดชบอร์ดและรายงาน

### 5.1 หน้าหลัก (Dashboard)

แสดง 4 ตัวชี้วัดหลัก:
- **แผนงานทั้งหมด** — จำนวนแผนในระบบของแผนกคุณ
- **แผนที่กำลังดำเนินการ** — แผนที่อยู่ในสถานะ ACTIVE
- **ความคืบหน้างานโครงการ (%)** — ค่าเฉลี่ยจากทุกงาน PROJECT
- **การปฏิบัติตามแผนงานประจำ (%)** — ค่าเฉลี่ย Compliance จากทุกงาน ROUTINE

### 5.2 ศูนย์รวมรายงาน (Reports Hub)

ไปที่เมนู **รายงาน** เพื่อเลือกดูรายงานแต่ละประเภท:

| รายงาน | เนื้อหา |
|---|---|
| **สรุปแผนงานประจำปี** | ภาพรวมจำนวนงานและ % ความสำเร็จแต่ละแผน |
| **ความคืบหน้างานโครงการ** | รายละเอียดความคืบหน้าและอัปเดตล่าสุดของงาน PROJECT |
| **การปฏิบัติตามงานประจำ** | % Compliance และวันที่ขาดหายของงาน ROUTINE |
| **การประเมินผล KPI** | เปรียบเทียบผลที่ทำจริงกับแผนสำหรับ Officer และ Supervisor |
| **ผลงานพนักงาน** | สรุปผลงานรายบุคคลของ Officer |
| **แผนงานประจำปี (Gantt)** | ตารางแผนวาดตามเวลา สามารถพิมพ์ PDF ได้ |

**การ Export:**
- กดปุ่ม **พิมพ์ / PDF** เพื่อพิมพ์หรือบันทึกเป็น PDF
- กดปุ่ม **Export Excel (CSV)** เพื่อดาวน์โหลดข้อมูล

## 6. เคล็ดลับการใช้งาน

> **สำหรับ Manager:** ลำดับการทำงานที่ถูกต้องคือ สร้างแผน → เพิ่มงาน → อนุมัติแผน → ติดตามผล

> **สำหรับ Officer:** เช็คหน้า "งานของฉัน" ทุกวัน โดยเฉพาะงานประจำที่ต้องส่งรายงานตามรอบ ไม่อย่างนั้นจะถูกนับเป็น Missed

> **การดูวันที่ขาด:** เปิดงาน ROUTINE → แท็บ "ปประวัติการบันทึก" → ระบบจะแสดงช่วงเวลาที่ยังไม่ได้บันทึก (สีแดง)

> **เปลี่ยนภาษา:** คลิกโปรไฟล์ → เลือก ภาษา → เลือก ລາວ / ไทย / English ได้ทุกเวลา

## 7. สถานะต่างๆ ในระบบ

### สถานะแผนงาน (Work Plan Status)

| สถานะ | ความหมาย |
|---|---|
| **ฉบับร่าง (DRAFT)** | กำลังเพิ่มงาน ยังไม่เปิดใช้งาน |
| **กำลังดำเนินการ (ACTIVE)** | อนุมัติแล้ว Officer ส่งผลงานได้ |
| **ปิดแล้ว (CLOSED)** | สิ้นสุดแผน ไม่รับข้อมูลใหม่ |

### สถานะงาน (Task Status)

| สถานะ | ความหมาย |
|---|---|
| **รอดำเนินการ (PENDING)** | ยังไม่เริ่มทำ |
| **กำลังดำเนินการ (IN PROGRESS)** | อยู่ระหว่างปฏิบัติงาน |
| **สำเร็จแล้ว (DONE)** | ดำเนินการเสร็จสมบูรณ์ |

### สถานะการส่งรายงาน (Actual Status)

| สถานะ | ความหมาย |
|---|---|
| **สำเร็จ (DONE)** | ส่งงานครบตามรอบ |
| **สำเร็จบางส่วน (PARTIAL)** | ส่งบางส่วน ยังไม่ครบ |
| **ไม่ได้ปฏิบัติ (NOT DONE)** | ไม่ได้ส่งรายงานในรอบนี้ |
